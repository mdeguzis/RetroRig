    <article class="markdown-body entry-content" itemprop="mainContentOfPage"><h1>
<a name="user-content-retrorig-project" class="anchor" href="#retrorig-project" aria-hidden="true"><span class="octicon octicon-link"></span></a>RetroRig Project</h1>

<p><a href="https://camo.githubusercontent.com/9da030e8f042e313b6d3a2867e9b71aeb7d83df0/687474703a2f2f692e696d6775722e636f6d2f497267763046782e706e67" target="_blank"><img src="https://camo.githubusercontent.com/9da030e8f042e313b6d3a2867e9b71aeb7d83df0/687474703a2f2f692e696d6775722e636f6d2f497267763046782e706e67" alt="alt text" title="RetroRig" data-canonical-src="http://i.imgur.com/Irgv0Fx.png" style="max-width:100%;"></a></p>

<h6>
<a name="user-content-version-091" class="anchor" href="#version-091" aria-hidden="true"><span class="octicon octicon-link"></span></a>[Version 0.9.1]</h6>

<p>RetroRig is shell script to setup a Linux x86_64 system  with several emulators, and XBMC as graphical
front end.The inspiration for doing this lies almost completely with the RetroPie Project. I wanted to provide
something similar, but with XBMC, x86_64, and my favorite interface, "Rom Collection Browser."
This project is intended to be run on Ubuntu (currently 14.04 LTS), and a gamepad / controller.
At some point in the future, I want to try and branch this to other distributions if possible. 
Please see the features page on the wiki for ideas and future plans.</p>

<p>I invite you to challenge the configs and scripts to help improve my ultimate goal to provide
an easy way to get up and running with RetroGaming on x86_64 Linux systems. Pull requests or
Issues are very much welcome! Please see the AUTHORS file in the root directory for all the awesome people
that contribute to this project.</p>

<p>You can follow RetroRig's development by staring the github.com page, or with joining the IRC channel <code>#retrorig-dev</code>
located under the irc.freenode.net network. The main op is me "ProfessorKaos64". You can also follow me on twitter with the username "@N3RD42". There is also a <a href="http://libregeek.org/forum/index.php">forum</a> for any and all discussion.</p>

<p><strong>Please be advised:</strong><br>
RetroRig requires sudo access to inject the xboxdrv 
init sripts, install softwate via apt-get, and other 
various items. All script code and config files are 
available for review.</p>

<p>Thank you for your patience.</p>

<h2>
<a name="user-content-demo-video" class="anchor" href="#demo-video" aria-hidden="true"><span class="octicon octicon-link"></span></a>Demo Video</h2>

<p><a href="https://www.youtube.com/watch?v=G49MMQi2zHE"><img src="https://camo.githubusercontent.com/7fe09a482bb3a318651517fde719bd8be12a23b7/687474703a2f2f692e696d6775722e636f6d2f43547251494b492e706e67" alt="ScreenShot" data-canonical-src="http://i.imgur.com/CTrQIKI.png" style="max-width:100%;"></a></p>

<h2>
<a name="user-content-warning" class="anchor" href="#warning" aria-hidden="true"><span class="octicon octicon-link"></span></a>Warning</h2>

<p>RetroRig is meant to be a standalone setup for XBMC on Ubuntu. Soon, RetorRig will use it's own dotfile so you can continue to use your own XBMC implementation. RetroRig now makes use of a custom-patched XBMC binary as well to allow hotplugging for support controllers is possible. It is mainly targeted at folks wishing to repurpose an old physical PC. VirtualBox support exists, (to an extent), but can behave erratcially with emulator speed/framerates and is not advised. I do hope to corrrect some of that in due time, but right now, it is more of an "extra" or a "test bed."</p>

<p>It is mainly targeted at folks wishing to repurpose an old physical PC. VirtualBox support exists, 
(to an extent), but can behave erratcially with emulator speed/framerates and is not advised. I do
hope to corrrect some of that in due time, but right now, it is more of an "extra" or a "test bed."</p>

<p>It will overwrite some folders for supported emulators and a few system files. Please see the 
wiki entries under "Installation," and "Advanced Configuration," or checkout the source code.</p>

<h2>
<a name="user-content-some-current-features-include" class="anchor" href="#some-current-features-include" aria-hidden="true"><span class="octicon octicon-link"></span></a>Some current features include</h2>

<p>For all other features and future plans, please see the <a href="https://github.com/ProfessorKaos64/RetroRig/wiki/Development-and-Features">wiki</a></p>

<ul class="task-list">
<li>No need to install an entire separate distro on your Mint/Debian machine</li>
<li>Custom XBMC binary package</li>
<li>Custom XBMC home screen and button layout</li>
<li>Seperate dot file (.retrorig, not .xbmc) so you can have your cake and eat it too!</li>
<li>Hotplugging support for supported wireles game controllers</li>
<li>Dual monitor support for XBMC</li>
<li>Auto-install software, emulator configs, and required components</li>
<li>Many supported consoles (more added frequently)</li>
<li>Gamepad select menu</li>
<li>SSH and samba share support</li>
<li>Preset Controller mappings for supported gamepads</li>
<li>Resolution presets/custom selection for emulators that support it</li>
<li>Save state, load state, exit emulators with gamepad</li>
<li>A cobbled together "first run" state of RCB with a blank games database and pre-set configs</li>
<li>Functions to update git repo, emulator binaries, upgrade system and more</li>
<li>Automatically start XBMC, then directly into RCB itself (option to boot to XBMC session in settings menu)</li>
<li>ROM pre-loader</li>
<li>BIOS pre-loader</li>
<li>Modular design for expandability</li>
<li>Unity configuration set during config-setup to lengthen screen timeout + remove screen lock</li>
</ul><h2>
<a name="user-content-wiki" class="anchor" href="#wiki" aria-hidden="true"><span class="octicon octicon-link"></span></a>Wiki</h2>

<p>First, I must direct you to read the wiki on this github page, as it contains extra iformation outside this wiki. 
<a href="https://github.com/ProfessorKaos64/RetroRig/wiki">https://github.com/ProfessorKaos64/RetroRig/wiki</a></p>

<p><strong>Please also note</strong><br>
You may add the <code>--help</code> flag to the script for a quick help file. Come on, do it, it has psychedelic colorrrrsss!</p>

<h2>
<a name="user-content-installation" class="anchor" href="#installation" aria-hidden="true"><span class="octicon octicon-link"></span></a>Installation</h2>

<p>Pre-requisites:</p>

<p>You will need git,figlet and dialog to run the installer:</p>

<pre><code>sudo apt-get install dialog git figlet
</code></pre>

<p>RetroRig will try to install them for you (for instance, if you download the zip archive), but if you
experience any issues starting the script,enusre they exist with the CLI commands 'which dialog' 
and 'which git" to ensure they report back. Please also ensure your graphics card or onboard graphics chipset supports OpenGL.</p>

<p>To clone this repo via the CLI:</p>

<p><code>git clone https://github.com/ProfessorKaos64/RetroRig</code></p>

<p>To install:</p>

<pre><code>cd RetroRig
sudo ./retrorig_setup.sh
</code></pre>

<p>Currently,Only Ubuntu 14.04 LTS is supported, but more will be added in due time! You can perform a <code>git checkout Beta</code> to try our Beta branch, but please do so with caution and backup any file before hand!</p>

<p>You can also download a zip file or by other means on the github page.</p>

<h2>
<a name="user-content-updating" class="anchor" href="#updating" aria-hidden="true"><span class="octicon octicon-link"></span></a>Updating</h2>

<p>The retrorig_setup.sh script also currently contains mechanisms to upgrade Ubuntu, update the
emulator binaries, update our patched XBMC version, and also a way to pull the latest files from github. </p>

<h2>
<a name="user-content-please-note" class="anchor" href="#please-note" aria-hidden="true"><span class="octicon octicon-link"></span></a>Please Note</h2>

<p>This project is not yet complete! Project notes and guide will be hosted at 
<a href="http://www.libregeek.org">www.libregeek.org</a> at some point in the near future.</p>
